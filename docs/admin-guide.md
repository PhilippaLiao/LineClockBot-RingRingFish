# LineClockBot Admin 後台介紹

## 概覽

Admin 後台是 LineClockBot 的管理介面，提供系統管理員查看與管理所有使用者資料、打卡紀錄及提醒的功能。

- 入口路徑：`/Admin/Login`
- 技術實作：ASP.NET Core Razor Pages
- 所有頁面共用 `_AdminLayout.cshtml` 版型，頂部導覽列可快速切換各功能頁面

---

## 登入驗證

### 登入方式

後台採用 **Session-based 驗證**，以 `Internal API Key` 作為密碼。

| 項目 | 說明 |
|------|------|
| 登入頁面 | `/Admin/Login` |
| 驗證方式 | 輸入 `Internal__ApiKey`（即環境變數 `Internal__ApiKey` 的值） |
| Session 有效期 | 8 小時 |
| 登出頁面 | `/Admin/Logout`（清除 Session，導回登入頁） |

### 驗證機制

所有 Admin 頁面皆繼承 `AdminPageBase`，每次請求都會執行 `CheckAuth()`：

```csharp
if (HttpContext.Session.GetString("AdminAuthed") != "true")
    return RedirectToPage("/Admin/Login");
```

未登入或 Session 過期時，會自動導向登入頁。

## 功能頁面

### 1. UserSettings（使用者設定）

**路徑：** `/Admin/UserSettings`

管理所有曾使用 Bot 的 LINE 使用者及其設定。

**顯示欄位：**

| 欄位 | 說明 |
|------|------|
| ID | 資料庫流水號 |
| 暱稱 | 可編輯的使用者暱稱 |
| Line User ID | LINE 平台的使用者識別碼 |
| 工時 | 該使用者設定的每日工作時數 |
| 更新時間 | 最後一次設定更新時間（台灣時間） |

**可執行操作：**

- **編輯暱稱**：直接在表格內修改暱稱並按「儲存」，方便在其他頁面以暱稱辨識使用者
- **刪除使用者**：刪除後該使用者需重新在 Bot 設定工時才能使用

> 注意：刪除使用者設定不會同步刪除其打卡紀錄與提醒，需分別至各頁面處理。

---

### 2. WorkRecords（打卡紀錄）

**路徑：** `/Admin/WorkRecords`

查看與管理所有使用者的打卡紀錄。

**顯示欄位：**

| 欄位 | 說明 |
|------|------|
| ID | 資料庫流水號 |
| Line User ID | 顯示暱稱（hover 可見原始 ID） |
| 上班時間 | 使用者打卡的時間 |
| 下班時間 | 依工時計算的預計下班時間 |
| 已通知 | 下班提醒是否已發送 |
| 通知時間 | 發送下班通知的時間 |
| 建立時間 | 紀錄建立時間 |

**可執行操作：**

- **使用者篩選**：下拉選單切換，只顯示特定使用者的打卡紀錄
- **單筆刪除**：點擊「刪除」按鈕，確認後刪除該筆紀錄
- **批次刪除**：勾選多筆後按「刪除選取」，一次刪除多筆紀錄
- **分頁瀏覽**：每頁顯示 30 筆，依打卡時間倒序排列

---

### 3. Reminders（提醒管理）

**路徑：** `/Admin/Reminders`

查看、新增與管理所有排程提醒。

**顯示欄位：**

| 欄位 | 說明 |
|------|------|
| ID | 資料庫流水號 |
| User | 顯示暱稱（hover 可見原始 ID） |
| 排程時間 | 預計發送提醒的時間 |
| 訊息 | 提醒內容文字 |
| Type | 提醒類型（目前只有 `custom` 使用者設定） |
| 狀態 | 待發送 / 已發送 |
| 已發送時間 | 實際發送時間（未發送顯示 `-`） |

**篩選功能：**

- **全部 / 待發送 / 已發送**：依發送狀態篩選
- **使用者篩選**：下拉選單切換特定使用者

**可執行操作：**

- **新增提醒**：填寫 Line User ID、日期、時間（HH:mm）及訊息內容後新增；系統會自動防止重複提醒
- **單筆刪除**：點擊「刪除」按鈕，確認後刪除
- **批次刪除**：勾選多筆後按「刪除選取」一次刪除
- **分頁瀏覽**：每頁顯示 30 筆

## 安全性說明

- Admin 後台**不對外公開**，依賴 `Internal API Key` 的保密性作為唯一驗證機制
- Session 設為 `HttpOnly` 及 `Essential`，防止 JavaScript 存取
- 所有敏感操作（刪除）皆有前端 `confirm()` 確認彈窗防止誤操作
- 每次請求都會重新驗證 Session，避免過期 Session 繼續存取
